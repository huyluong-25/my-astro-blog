Chào mọi người, mình là **Huy**. Đây là bài blog đầu tiên trên "ngôi nhà mới" của mình. Với một người làm IT Support, việc dựng được hệ thống CI/CD với Jenkins và Docker cho blog cá nhân thực sự là một trải nghiệm rất thú vị.

## 1. Một ngày làm việc tại TiniGroup

Công việc hàng ngày của mình xoay quanh việc hỗ trợ kỹ thuật, quản lý hệ thống mạng và xử lý các ticket. Có những ngày khá "căng" với các lỗi về DHCP hay cấu hình Firewall, nhưng đó cũng là lúc mình học được nhiều nhất.

> "Làm IT không chỉ là sửa máy, mà là giải quyết vấn đề cho con người."

## 2. Góc học tập: Từ vựng IT & TOEIC

Vì đang trong quá trình luyện thi TOEIC và nâng cao vốn tiếng Anh chuyên ngành, mình thường ghi chú lại các từ vựng hay gặp:

| Từ vựng | Nghĩa | Ví dụ |
| :--- | :--- | :--- |
| **Troubleshoot** | Xử lý sự cố | I need to troubleshoot the DNS issue. |
| **Deployment** | Triển khai | The deployment on Jenkins was successful. |
| **Authentication** | Xác thực | Please check your login authentication. |

## 3. Lệnh Git "thần thánh" mình vừa học

Để đẩy code lên server nhanh nhất, mình thường dùng dòng lệnh gộp này:

```bash
git add . && git commit -m "Update blog" && git push
```

Lệnh này giúp tiết kiệm thời gian khi làm việc cá nhân, nhưng trong môi trường team thì mình vẫn tách từng bước để dễ kiểm soát lịch sử commit.
